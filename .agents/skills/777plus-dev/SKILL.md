# 777plus Development

在本项目发生开发性修改时使用本 skill。

## 开始前

1. 读取根目录 `AGENTS.md`、`spec.md` 和 `README.md`。
2. 检查 Git 状态、分支、未提交改动、目录结构、运行入口和验证入口。
3. 从当前任务涉及的入口追踪真实规则、状态、数据、错误和外部边界。
4. 区分已确认事实、owner 提供但未验证的事实、推测和已完成并验证事项。

## 实现规则

- 优先沿用仓库已经成立的技术栈、命名、模块边界和本地辅助能力。
- 状态、规则、数据读写、展示、外部接线和错误处理变化原因不同时，拆清边界。
- 结构化数据使用结构化工具处理，错误在拥有处理能力的边界解决。
- 不覆盖或回滚 owner 的无关改动，不建立旧别名、兼容转发或过渡文件。
- 任何未实现或未验证内容，在任何载体中都不能写成已经实现或已经通过。

## 内容规则

- Markdown 原材料保留原文事实，不因排版精修改变含义。
- 重点词映射必须来自人工确认的真实原文；找不到依据时保留空映射。
- 通用搜索与人工语义重点是两个独立能力，不混用数据来源。
- 不重新运行素材转换脚本覆盖已经人工精修的 Markdown，除非 owner 明确要求重新转换。

## 测试与文档

- 测试保护真实规则、边界、协作合同和核心路径，不保护内部实现形状。
- 产品当前事实变化时同步 `spec.md`。
- 用户入口、运行或验证方式变化时同步 `README.md`。
- 长期协作规则或验证入口变化时同步 `AGENTS.md` 和本 skill。

## 完整验证

- `npm run verify`
- `python -X utf8 start_index.py --check`

先运行与改动最接近的检查，再运行完整验证。无法运行时写明具体命令、阻塞原因和剩余风险。

## 交付

- 检查最终差异、文件清单和工作区状态，只交付当前任务相关内容。
- commit、push、部署和发布只在 owner 明确授权时执行。
- 收口说明完成事实、实际验证、未验证项和剩余风险。
