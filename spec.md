# 777plus Spec

`spec.md` 只记录当前生效的产品事实与边界。

## 产品定位

- 面向考试备考人员的纯前端知识看板，用于集中阅读工作材料、定位关键词原文上下文，并组织论述题与自拟题答题结构。

## 当前事实

- 项目名称：`777plus`。
- Git 默认分支为 `master`，远端为 `https://github.com/luckymaomi/777plus.git`。
- 原始素材位于 `考试素材合集/`，目录当前有 13 份 `.doc`、`.docx`、`.pdf` 和 `.pptx` 文档。
- 看板采用其中 11 份有效文献，已整理为 `content/materials/*.md`，目录与来源信息位于 `content/catalog.json`。
- 两份 2024 年材料不再进入看板；两份 2025 年材料保留并标注为“旧材料”。
- 主导航包含文献综述、关键词和答题模板三个模块，并提供通用搜索、主题切换和移动端导航。
- 顶部更新横幅以不可点击的纯文本呈现当前更新范围与日期。
- 根目录 `start_index.py` 构建并启动本地站点，只通过浏览器匿名模式打开页面。
- 首页首次进入时显示复习指引，说明文献综述、关键词和答题模板的使用顺序；用户可关闭自动显示并从顶栏重新打开。

## 核心用户路径

1. 用户进入看板后，从文献综述、关键词或答题模板模块开始。
2. 用户选择材料阅读原文，或选择关键词查看其在不同材料中的原文上下文。
3. 用户按来源回看原文，并参考带原文引文的完整示例答案组织与本岗位相关的作答。

## 业务边界

- 当期包含文献综述、关键词定位和答题模板三类内容。
- 关键词只展示真实材料中检索到的上下文；找不到时留空，不生成解释。
- 关键词映射由人工通读材料后整理，不由通用搜索自动生成；重点页展示命中段落及相邻上下文。
- 重点上下文可以回到完整素材的对应段落，页面滚动定位并高亮该段落。
- 答题模板按安全、改革、绩效、党建融入中心和飞行队伍建设组织；每个模板包含基于原文撰写的多段论述、可核对的原文引文和原文定位。
- 答题模板只展示具体示例，不提供输入框、草稿存储或空白答案骨架。
- 文献综述阅读页不提供文内目录。
- 原始文献文字不删改；图片不迁移到看板。
- 素材会变化，内容放在 Markdown 中，稳定的页面逻辑不与材料正文耦合。

## 技术边界

- 采用纯前端架构、Bootstrap 视觉体系和 marked Markdown 渲染。
- 不需要服务端、数据库或真实浏览器自动化测试。
- 使用 Vanilla TypeScript、Vite、Bootstrap、marked、DOMPurify、Lucide 和 Vitest。
- 使用少量自动测试保护语义映射、搜索、路由和答题模板规则，不包含浏览器自动化测试。
- GitHub Actions 从 `master` 构建并部署 GitHub Pages。

## 数据边界

- 页面数据来自仓库内 Markdown 和结构化索引文件。
- 原始文件保留在 `考试素材合集/`，生成内容保留来源文件名与 SHA-256，用于核对材料版本。
- 项目不上传用户输入，不建立账号或跨设备同步。

## 当前不做

- 不迁移原文中的图片。
- 不为没有来源命中的关键词补写定义、上下文或结论。
- 不把需求说明、功能介绍或使用教学作为看板正文展示。
- 不实现后端、账号体系、云同步或重型端到端测试。

## 验收标准

- 11 份有效文献均可从目录进入并渲染文字内容。
- 关键词结果可追溯到具体素材和原文上下文；无结果时明确为空。
- 文献综述、关键词和答题模板在桌面端与移动端均可使用。
- 前端完整验证命令 `npm run verify` 通过。
- 文档、实现、测试和配置描述同一套当前事实。
